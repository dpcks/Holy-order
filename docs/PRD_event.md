# Holy-Order PRD
## 이벤트(골든벨) 기능

---

## 1. 배경

Holy-Order는 교회 카페 운영을 위한 QR 주문/관리 시스템이다.

교회 카페 운영 특성상 특정 날에 교인 또는 가정이 후원하여
“오늘은 무료로 운영됩니다” 형태의 이벤트가 발생할 수 있다.

예시:
- 칠순 감사
- 결혼 감사
- 출산 감사
- 임직 감사
- 기타 감사/기념

이 기능은 위와 같은 이벤트를 시스템적으로 지원하기 위한 것이다.

---

## 2. 목적

이벤트(골든벨) 기능의 목적은 다음과 같다.

- 관리자 화면에서 이벤트를 생성/수정/활성화/종료할 수 있다.
- 사용자 화면에 이벤트 배너와 웰컴 모달을 보여줄 수 있다.
- 이벤트 중에는 모든 메뉴가 0원으로 표시된다.
- 기존 가격은 회색 + 취소선으로 표시된다.
- 주문은 무료로 접수되지만, 정산을 위해 원래 가격 기준 합계를 보관한다.
- 후원자 기준 정산 리포트를 제공한다.

---

## 3. 핵심 사용자 경험

### 3-1. 사용자 화면

이벤트 모드가 활성화되면:

1. 홈 화면 상단에 이벤트 배너 표시
2. 앱 진입 시 웰컴 모달 표시
3. 메뉴 가격이 무료(0원)로 표시
4. 기존 가격은 회색 + 취소선 처리
5. 장바구니 및 최종 결제 금액도 0원 처리

### 3-2. 관리자 화면

관리자는:
- 이벤트/공지 생성
- 배너 문구 입력
- 이미지 등록
- 후원자 정보 입력
- 이벤트 활성화 / 종료
- 정산 리포트 조회

---

## 4. 가격 표시 규칙

이벤트 모드일 때 가격은 반드시 아래 형식을 따른다.

- 기존 가격: 회색 처리
- 기존 가격: 가운데 줄(취소선)
- 실제 가격: 0원
- 실제 가격은 강조

예시:
~~3,500원~~ 0원

이 규칙은 아래 페이지에 일관되게 적용되어야 한다.

- Home.tsx
- MenuDetail.tsx
- Cart.tsx

---

## 5. 기능 범위

### 포함
- 이벤트/공지 생성
- 이벤트 활성화/비활성화
- 사용자 상단 배너
- 사용자 웰컴 모달
- 가격 0원 처리
- 원가 기준 정산 리포트
- 후원자 정보 저장

### 제외
- 여러 이벤트 동시 활성화
- 메뉴별 선택 이벤트
- 후원 결제 연동
- 자동 이미지 최적화 고도화
- 푸시 알림

---

## 6. 데이터 저장 원칙

이벤트 중 주문이 들어오면:
- 실제 사용자 결제 금액은 0원으로 저장
- 원래 가격은 별도로 저장
- 해당 주문이 어떤 이벤트에 속하는지 연결한다

즉:
- `total_price = 0`
- `original_price = 원래 계산 금액`
- `announcement_id = 활성 이벤트 ID`

---

## 7. 사용자 노출 방식

### 상단 배너
예시:
오늘은 김철수 장로님의 칠순 감사로 카페가 무료 운영됩니다.

### 웰컴 모달
표시 요소:
- 이벤트 이미지
- 제목
- 상세 문구
- 후원자 관련 메시지

### 장바구니
- 상품 금액
- 이벤트 할인
- 최종 결제 금액 0원

---

## 8. 관리자 정산 리포트

리포트에서는 아래 정보를 확인할 수 있어야 한다.

- 총 주문 건수
- 총 제공 수량
- 원래 금액 기준 총합
- 메뉴별 판매 수량 / 금액
- 직분별 통계

예시 문구:
오늘 총 120잔의 음료가 나눔 되었으며,
후원하실 총 금액은 425,000원입니다.

---

## 9. 구현 우선순위

이벤트 기능은 현재 신규 기능 중 최우선으로 진행한다.

이유:
- Holy-Order만의 차별화 포인트
- 교회 운영 문화에 매우 잘 맞음
- 사용자 경험 변화가 큼
- 실제 운영 만족도가 높을 것으로 기대됨

---

## 10. 기술 방향

- 기존 FastAPI + React + TypeScript 구조 유지
- 기존 `schemas.py` ↔ `types/index.ts` 대응 유지
- 이벤트 관련 데이터는 별도 모델로 관리
- 이미지 업로드는 초기에는 URL 기반 또는 단순 업로드 허용
- 추후 Cloudinary 확장 가능하도록 구조 유지
