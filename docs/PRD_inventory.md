# Holy-Order PRD
## 재고 관리 기능

---

## 1. 배경

현재 메뉴 품절 여부는 수동으로 관리하고 있지만,
실제 교회 카페 운영에서는 메뉴보다 재료/소모품 재고 파악이 더 중요하다.

예시:
- 파우더
- 우유
- 시럽
- 일회용컵
- 종이컵
- 뚜껑
- 빨대
- 냅킨

---

## 2. 목적

이 기능의 목적은 다음과 같다.

- 재료 및 소모품의 현재 재고를 기록한다.
- 기준 수량 이하인 재고를 빠르게 확인한다.
- 구매가 필요한 항목을 한눈에 파악한다.
- 관리자가 수기로 재고를 수정할 수 있게 한다.

---

## 3. 범위

### 포함
- 재고 항목 CRUD
- 카테고리 분류
- 현재 재고 수량
- 기준 수량
- 재고 부족 목록
- 관리자 재고 관리 페이지

### 제외
- 자동 차감
- 레시피 연동
- 자동 품절 처리
- 입출고 이력
- 발주 자동화

---

## 4. UX 요구사항

관리자 재고 페이지에서:
- 부족 재고 요약 카드
- 부족한 항목 리스트
- 전체 재고 목록
- 검색/카테고리 필터
- 추가/수정 모달 제공

상태 기준:
- current_stock > alert_threshold → 정상
- current_stock <= alert_threshold → 구매 필요

---

## 5. 우선순위

이 기능은 이벤트 기능 이후에 구현한다.

---

## 6. 기술 방향

- 기존 FastAPI + React + TypeScript 구조 유지
- schemas.py ↔ types/index.ts 대응 유지
- 단순 CRUD 중심
- 재고 관리는 수동 입력 기반
